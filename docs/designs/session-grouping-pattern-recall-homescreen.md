## Vision

### 10x Check
A bottom sheet that doesn't just show raw past instances — it renders a glucose curve overlay where all past curves for this meal are superimposed on a single chart. You can see at a glance whether this meal is consistent (tight curves) or variable (spread curves). A single visual answer to "is this meal predictable for me?" That's the 10x version.

### Context
This plan was submitted as a pre-GSD document. Phases 1–3 (session grouping, matching engine, history UI) are already built and committed. The review identified what's genuinely missing vs what already exists.

## Accepted Scope (added to this plan)
- Bottom sheet component (new) with up to 10 tabs per past instance — replaces ExpandableCard inline expand/collapse
- Averaged stats panel on MealLogScreen showing avgRise/avgPeak/avgTimeToPeak above individual match rows
- SafetyDisclaimer reusable component (required on all screens showing pattern data)
- HomeScreen visual redesign: Outfit + JetBrains Mono fonts, #050706 background, arc gauge, pulsing LIVE indicator, SVG icons, staggered entrance animations
- theme.ts shared constants file: all colors, font names, font weights — used by HomeScreen now, all screens adopt over time

## Deferred to TODOS.md
- Variable session windows (60/90/180min by meal type): keyword matching is too fragile for production use. Revisit when 50+ meals logged and meal naming patterns are observable. A classifier trained on real data (not a keyword list) is the right approach.
